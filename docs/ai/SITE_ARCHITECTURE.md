## Page Architecture

### Home
- Objective: Convert local visitors into WhatsApp contacts; capture company uniform leads as secondary.
- Sections:
  1. Header (sticky) — wordmark "Valdete Costuras" + WhatsApp CTA.
  2. Hero — headline (what + where) + subheadline + WhatsApp CTA + experience signal (`[ANOS]` anos).
  3. Serviços — alterations/repairs grid (hems, zippers, resizing, repairs, etc.).
  4. Diferenciais — 4 pillars: experiência · rapidez · atendimento pessoal · preço justo.
  5. Como funciona — 3 steps: traga a peça → orçamento na hora → pronto rápido.
  6. Empresas / Uniformes — B2B block with dedicated "orçamento para empresas" CTA.
  7. Galeria (antes/depois) — placeholder grid; hidden or "em breve" while empty.
  8. Depoimentos — placeholder; hidden or "em breve" while empty.
  9. Localização e horário — address `[ENDERECO]`, map/embed, hours `[HORARIO]`.
  10. CTA final — full-width band with strong WhatsApp CTA.
  11. Footer — brand, WhatsApp, address, hours, social (if any).
- Primary CTA: WhatsApp (`[WHATSAPP]`).
- Notes: Sections 7 and 8 must not break layout while empty; treat as progressive enhancement.

## Conversion Path
Entry → Hero value + location → confirm service → trust pillars → how it works → WhatsApp. Company path: Hero → Empresas section → company WhatsApp CTA.

## SEO / Accessibility / Performance Requirements
- Local-business intent: title/description include service + location; consider `LocalBusiness` structured data once address/phone are real.
- One H1 (hero). Logical heading order per section.
- WCAG AA contrast; keyboard navigable anchors; visible focus.
- Optimize with `next/image`; preload/subset fonts via `next/font`.
- Target strong LCP (hero text-first helps), low CLS, responsive INP.

## Asset Guidelines
- Use optimized images.
- Prefer SVG for logos and icons.
- Avoid embedded text inside images when content should be indexed or translated.
- Define meaningful alt text.
- Keep video and 3D assets conditional and performance-aware (none planned here).
