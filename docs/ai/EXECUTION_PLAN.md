## Stack
Next.js (App Router) + Tailwind CSS + TypeScript. `next/font/google` for Space Grotesk + IBM Plex Sans. Single route `/` composed of section components under `components/sections/`. Central `content` config (e.g. `lib/content.ts`) holding copy + placeholders so text is easy to edit.

## How to use skills in each phase
Each phase names a recommended primary skill. Before invoking any skill:
1. Confirm the skill is available (status `installed` in `catalog/approved-skills.md`).
2. If `installed`, propose it, state expected output, and ask approval before running.
3. If not installed (`target`), fall back to built-in guidance in `references/` and tell the developer which skill would have improved this phase.
Never run two overlapping visual skills in the same phase (see `catalog/skill-conflicts.md`).

## Phase 01 — Project foundation
### Goal
Scaffold Next.js + Tailwind + TS. Configure fonts (`next/font`), Tailwind theme tokens (colors/spacing/radius from `DESIGN_SYSTEM.md`), global styles, base layout, container, and the content config file. Set metadata (title/description) and viewport.
### Recommended skill
`react-best-practices` (fallback: `references/stack-selection.md`).
### Execution Prompt
Read all required docs. Diagnose before editing. Implement only this phase. Report changed files, decisions, and pending issues.

## Phase 02 — Essential components
### Goal
Build the core component set from `DESIGN_SYSTEM.md`: Button (primary/secondary/WhatsApp), Navbar (sticky), Footer, SectionHeader (eyebrow+H2), ServiceCard, Step, Placeholder block, FloatingWhatsApp. Wire WhatsApp CTA helper (`wa.me/[WHATSAPP]` with encoded message).
### Recommended skill
`composition-patterns`; `design-system` if tokens need structure.

## Phase 03 — Page structure
### Goal
Assemble all 11 sections per `SITE_ARCHITECTURE.md` + `PAGE_BLUEPRINTS.md`, driven by the content config. Gallery/testimonials render graceful "em breve" placeholders. Anchor navigation working.
### Recommended skill
`frontend-design` for fast structural implementation (polish comes in Phase 04).

## Phase 04 — Visual direction pass
### Goal
Apply Costura Moderna craft: type scale, spacing rhythm, accent usage, hairlines, hover states, contrast per `DESIGN_SYSTEM.md`. Make the hero feel premium.
### Recommended skill
`impeccable` as primary visual executor. Do NOT also run `frontend-design` or `ui-ux-pro-max` on the same decision.

## Phase 05 — Motion and interactions
### Goal
Implement motion per `MOTION_DIRECTION.md`: hero stagger, scroll reveals, microinteractions, floating WhatsApp. Respect `prefers-reduced-motion`. No CLS.
### Recommended skill
`impeccable` motion. No WebGL/3D. Respect `references/motion-principles.md`.

## Phase 06 — Responsive pass
### Goal
Validate 320px → desktop. Fix overflow, tap targets, grid reflow, floating button behavior, hero clamp.
### Recommended skill
`webapp-testing`.

## Phase 07 — Quality pass
### Goal
Accessibility (semantics, focus, contrast — verify accent), SEO (title/desc, one H1, LocalBusiness data once real contact exists), performance (images/fonts, Core Web Vitals), no visible placeholders shipped. Gate against `QA_CHECKLIST.md`.
### Recommended skill
`web-design-guidelines` for UI/UX/accessibility; `react-best-practices` for performance.

## Optional — Deploy
After QA approval only: `deploy-to-vercel`.
