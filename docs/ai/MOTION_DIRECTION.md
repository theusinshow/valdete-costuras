# Motion Direction

> Generated by Coded by M — Site Architecture System. Source of truth: `/docs/ai/DISCOVERY.md` and `/docs/ai/DECISIONS_LOG.md`.

## Core Motion Concept
Motion is quiet and precise — like a clean seam. It guides attention and confirms interaction; it never performs. Coherent with the minimal-premium, high-contrast direction. Nothing bounces or spins.

## Motion Language
- Easing: `cubic-bezier(0.22, 1, 0.36, 1)` (ease-out) for entrances; `ease` for hovers.
- Durations: micro 120–180ms; reveals 400–600ms.
- Distance: small (8–16px translate). Prefer opacity + slight translateY.
- No parallax, no scroll-jacking, no auto-playing video.

## Hierarchy
1. **Primary narrative motion** — hero elements fade/rise in on load (staggered: eyebrow → H1 → sub → CTA).
2. **Section transitions** — each section reveals once on scroll into view (IntersectionObserver), subtle fade + translateY, staggered children in grids.
3. **Microinteractions** — button hover (accent deepen to `--accent-strong`, tiny lift), card hover (lift + accent hairline), link underline, focus ring.
4. **Decorative motion** — minimal; optional thin accent line that draws in under section titles.

## Page / Section Priorities
- Hero: highest craft — staggered entrance.
- Serviços / Diferenciais / Como funciona: staggered card reveals.
- Empresas / Localização: simple single reveal.
- Final CTA: gentle emphasis (fade + subtle scale from 0.98).
- Gallery/testimonials placeholders: no motion while empty.

## Microinteractions
- Buttons: bg `--accent` → `--accent-strong`, `translateY(-1px)`, shadow-sm → hover lift, 150ms.
- Cards: `translateY(-2px)` + reveal accent hairline top border, 180ms.
- Sticky/floating WhatsApp button: subtle scale on tap; gentle appear after hero.
- Links: underline expands from left, 150ms.

## Mobile Behavior
- Reduce reveal distance and stagger; keep durations short (≤400ms) for perceived speed.
- Floating WhatsApp button fades in after hero scrolls out.
- No hover-dependent affordances; ensure tap feedback.

## Accessibility
- Respect `prefers-reduced-motion: reduce` → disable reveals/translate, keep instant opacity, keep focus states. All content must be fully usable with motion off.

## Performance Restrictions
- Do not compromise LCP, CLS or INP. Hero text must render immediately (no opacity:0 trap without fallback — reserve space, animate transform only).
- Use CSS transitions + IntersectionObserver, or a lightweight approach; avoid heavy animation libraries unless justified.
- No WebGL / 3D — not warranted for this project.
- Prevent CLS: reveals animate transform/opacity only, never layout properties.
